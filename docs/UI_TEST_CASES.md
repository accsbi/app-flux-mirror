# UI テストケース（全ゲーム共通）

ここで起きる不具合は**特定ゲーム固有ではなく、全カードゲームで共通に起こりうる**UIの観点。
新規実装・修正後は必ずこの一覧で確認する。確認の証跡（スクショ＋判定ログ）は
`screenshots_log/<game>/` に **同名ペア**（`<name>.png` ＋ `<name>.log`）で残す。
**スクショを1枚撮ったら必ず同名の判定ログ(.log)を書く**（例: `hoge-modal.png` ＋ `hoge-modal.log`）。

対象ゲーム: blackjack / poker / casino-war / high-low / old-maid / memory-battle

最終更新: 2026-06-22

---

## 1. レイヤ / 重なり（Z-order & overlap）

| ID | 観点 | 期待結果 | 確認方法 |
|---|---|---|---|
| Z01 | chrome モーダル（設定/ガイド/確認/コイン補充）表示中 | 盤面の装飾（席マーカー・カード・順位・メッセージ）が**モーダルに重ならない**。モーダルが最前面で操作可能。 | 各モーダルを開き、四隅・中央が他部品に被られていないか目視。 |
| Z02 | ゲーム自身のダイアログ（親決め/整理/ポーズ等） | **半透明**で下の盤面（席アバター/カード）が透ける。暗幕で全消ししない（手本準拠）。 | ダイアログ表示中に背後の部品が見えるか。 |
| Z03 | ヘッダー（COIN/BET・ツールバー） | 盤面カード・モーダル・マーカーと**重ならない**。常に最上段に独立表示。 | 各フェーズでヘッダー帯を目視。 |
| Z04 | フッター（FEEDBACK 等） | 盤面・モーダル・手札と**重ならない**。最下段に独立表示。 | 各フェーズでフッターを目視。 |
| Z05 | 状況メッセージ文字 | 常に**最前面**で読める。カード/捨て札に埋もれない（必要なら背面に帯）。 | メッセージがカードに重なる局面で可読性を確認。 |
| Z06 | 最前面の盤面装飾（席マーカー等） | chrome モーダルより**下**、ゲームダイアログ（半透明）より**下**。盤面カードより上。 | モーダル開閉で前後関係を確認。 |

## 8. 証跡（screenshots_log）の運用 — 必須ルール

- 置き場所: `screenshots_log/<game>/`（ゲームごとにフォルダ）。
- **スクショを1枚撮ったら、必ず同名の判定ログを書く**（1対1・例外なし）。
- ペア: `<name>.png`（スクショ）＋ `<name>.log`（テスト計画・確認項目・**判定 PASS/FAIL**）。**ベース名を一致**させる（例: `hoge-modal.png` / `hoge-modal.log`）。
- **最終チェックは人間が行う。** AI/スクリプトの PASS はあくまで一次判定。
- `screenshots_log/<game>/human_checked/` は **人間専用フォルダ**。人間が最終確認したものだけを人間が移動する。
  - **AI・スクリプト・自動処理は `human_checked/` に一切触れない**（書き込み・移動・削除・リネーム禁止）。
  - 同様に、自分が作っていないファイル/フォルダ（ユーザー作成物）は勝手に削除・改変しない。
- `.log` の雛形:

```
# <name>  （対象フェーズ/シナリオ）
- 関連テストID: Z01, PH02, SEAT01 ...
- 手順: メニュー→…→この画面
- 期待: （箇条書き）
- 実際: <name>.png 参照
- 判定: PASS / FAIL（理由）
```
