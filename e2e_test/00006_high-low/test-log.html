② 実機テストで発覚した不具合と修正（独自実装が原因）

BUG-1（音ならない＋START/ガイド等が無反応）：high-low が共通 buildGameAssetUrl/loadAppConfig を使わず独自解決器を持ち、その Android 分岐が旧 ./assets/... のまま＝ページ /assets/high-low.html 基準で /assets/assets/... の二重 assets で 全404。config・BGM・カード画像が読めず無音＆無反応（設定/広告削除は config 不要なので反応）。
FIX-1：high-low-config.ts / high-low-assets.ts の Android 専用分岐を削除し、共通と同じ base 相対 ${BASE_URL}web-games/game-assets/... に統一（Android base ./ で /assets/web-games/game-assets/... に解決）。WEB 分岐は不変。→ レンダーで START/ガイド/カード/BGM すべて 0×404 を確認（下表 C 群）。
BUG-2（別のカードゲーム・お知らせ更新が無反応）：メニュー部品は menu-news/menu-other-games イベントを発火するだけだが、high-low 独自 standalone はこのハンドラとモーダルが丸ごと欠落（共通 standalone は onNews/onOtherGames＋モーダルを持つ）。
FIX-2：共通部品 news-info-modal-panel / other-games-modal-panel を high-low に配線（@menu-news/@menu-other-games＋state＋card-games-list.json 取得＋android back）。コピペでなく共有部品の再利用＝old-maid と同一UX。→ 両モーダルが開くことをレンダーで確認（下表 B 群）。
